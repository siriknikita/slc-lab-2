# Документ вимог до продукту (PRD)

## 1. Мета продукту

Десктопний застосунок для формування реліз-повідомлень у Discord на основі даних з Jira-тікетів. Користувач вводить список ідентифікаторів або посилань на Jira-тікети, отримує зведені дані по ним та генерує готовий текст реліз-повідомлення для публікації в каналі Discord.

## 2. Цільові користувачі

- Розробники програмного забезпечення
- Реліз-менеджери та координатори релізів
- Учасники команд, які ведуть облік роботи в Jira та інформують спільноту через Discord

## 3. Функціональні вимоги

### 3.1 Введення тікетів

- Можливість ввести список Jira-тікетів у вигляді:
  - ключів (наприклад, `PROJ-1`, `PROJ-2`);
  - або placeholder-посилань / ідентифікаторів для подальшої підтримки реальних посилань.
- Підтримка введення будь-якої кількості тікетів (список).

### 3.2 Отримання даних по тікетах

- Функція «отримання» даних по введених тікетах.
- У поточній реалізації (навчальний мок) використовуються лише мок-дані без звернення до реального Jira API.
- Користувач має бачити результат у вигляді зведеної інформації по тікетах (наприклад, ключ, назва, опис або статус).

### 3.3 Генерація реліз-повідомлення для Discord

- Автоматична генерація тексту реліз-повідомлення за шаблоном на основі даних по тікетах.
- Повідомлення призначене для публікації в Discord-каналі (формат тексту, зручний для читання в чаті).

### 3.4 Копіювання та експорт

- Можливість скопіювати згенероване реліз-повідомлення в буфер обміну (кнопка «Копіювати»).
- Відображення готового тексту для ручного копіювання або перегляду.

## 4. Нефункціональні вимоги

### 4.1 Технологічний стек

- **Backend / оболонка:** Tauri (Rust) — десктопний застосунок.
- **Frontend:** React з Vite.
- **Стилізація:** Tailwind CSS.
- **Компоненти UI:** ShadCN UI (адаптація під Tauri/React).

### 4.2 Локалізація інтерфейсу

- Підтримка двох мов інтерфейсу: англійська (EN) та українська (UK).
- Вибір мови здійснюється в налаштуваннях застосунку (Preferences) та зберігається між сесіями.

### 4.3 Платформа

- Десктопні ОС, що підтримуються Tauri (Windows, macOS, Linux).

## 5. Обмеження та припущення

- **Без реальних зовнішніх API:** у рамках навчального проєкту не використовуються реальні Jira API та Discord API. Усі дані по тікетах та згенероване повідомлення базуються на моках.
- **Мок-дані:** достатньо набору фіксованих або простых мок-відповідей, щоб продемонструвати сценарій «введення тікетів → отримання даних → генерація повідомлення».
- Документ описує початковий (базовий) обсяг для демонстрації підходу Waterfall; розширення функціоналу (реальна інтеграція з Jira/Discord) може бути визначено окремо.

## 6. Артефакти планування

- Цей PRD є вхідним документом для етапу «Планування та вимоги» методології Waterfall.
- Узгоджується з документом Project Baseline (обсяг, графік, ресурси та бюджет).
